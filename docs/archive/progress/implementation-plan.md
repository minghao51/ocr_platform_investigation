## Performance Benchmarks

### Targets
- Single image processing: < 30s
- 10-page PDF processing: < 2min
- Page load time: < 2s
- API response time (non-VLM): < 500ms
- Database query (indexed): < 100ms

### Optimization Strategies
1. **Async operations** - Concurrent page processing
2. **Image optimization** - Resize before API call
3. **Database indexing** - All filter columns indexed
4. **Frontend lazy loading** - Virtual scrolling for large tables
5. **Docker caching** - Layer caching for faster rebuilds

## Risk Mitigation

### Known Risks
1. **VLM API rate limits**
   - Mitigation: Optional rate limiting config
   - Fallback: Queue system if needed

2. **PDF processing memory**
   - Mitigation: Process pages in batches
   - Monitoring: Memory usage tracking

3. **SQLite performance**
   - Mitigation: Indexes, periodic VACUUM
   - Escape hatch: Switch to PostgreSQL if needed

4. **Docker image size**
   - Mitigation: Alpine base, multi-stage build
   - Target: < 2GB final image

5. **Schema validation complexity**
   - Mitigation: Clear error messages
   - Documentation: Schema writing guide
